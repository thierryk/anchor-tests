Accessible Name

All links but the first one are given an Accessible Name via various techniques (highlighted below).

  1. <a href="#">
    <svg focusable="false" ...> ... </svg>
</a>

    This link does not have an Accessible Name. In VoiceOver (VO) it is announced as "unlabelled image [filename]" ([filename] being "accessible-name.html" for this document).

    The rotor menu in VO displays "accessible-name.html".

  2. <a href="#">
    <svg focusable="false" role="image" ...> 
        <title>Anchor Link #2</title>
        ... 
    </svg>
</a>

    The rotor menu in VO displays "accessible-name.html" in Safari and Firefox (MacOS).

    The title here creates a "tooltip".

  3. <a href="#" aria-label="Anchor Link #3">
    <svg focusable="false" aria-hidden="true" ...> 
        ... 
    </svg>
</a>

    Relying on aria-label is not the best choice when it comes to Internationalization

  4. <a href="#">
    <svg focusable="false" role="image" title="Anchor Link #4" ...>
        ... 
    </svg>
</a>

    Safari and Firefox (MacOS) announces this link as "image accessible-name.html".

    The rotor menu in VO displays "accessible-name.html" in Safari.

    The title here does not create a "tooltip".

  5. <a href="#" title="Anchor Link #5">
    <svg focusable="false" role="image" ...>
        ... 
    </svg>
</a>

    The title here creates a "tooltip".

  6. Anchor Link #6 
    <a href="#">
    <svg focusable="false" aria-hidden="true" ...>
        ... 
    </svg>
    <span class="screen-reader-only">Anchor Link #6</span>
</a>
  7. <a href="#" class="css-generated-content">
    <svg focusable="false" aria-hidden="true" ...>
        ... 
    </svg>
</a>

    The rotor menu in VO displays "accessible-name.html" in Chrome.

    Relying on CSS generated content is not the best choice when it comes to Internationalization.

  8. <a href="#">
    <svg  role="image" aria-label="Anchor Link #8" focusable="false" ...> 
        ... 
    </svg>
</a>

    Relying on aria-label is not the best choice when it comes to Internationalization

Note: it is also possible to use aria-labelledby to reference a node containing the label (pointing to a heading's id for example).
